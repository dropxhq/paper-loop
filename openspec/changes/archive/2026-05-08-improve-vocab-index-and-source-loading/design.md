## Context

当前词表仅支持固定顺序展示，词量增长后难以快速定位；同时原文回链在弱网/HTML 不可用场景下缺少显式加载与失败反馈，用户容易感知为“点击无响应”。该变更跨越 `VocabView`、`SourceDetailView`、`HTMLReaderView`、`PDFReaderView` 与导入链路中的 anchor 语义，属于跨模块一致性改造。

约束：
- iOS SwiftUI MVP，必须保持现有 `Theme` 设计体系。
- 不引入后端改动；仅客户端行为与 UI 状态机优化。
- 兼容已有词卡与 occurrence 数据，避免破坏历史导入数据。

## Goals / Non-Goals

**Goals:**
- 提供 iPhone 词表右侧字母索引（A-Z + #）并支持滑动快速定位。
- 统一“打开原文”加载体验，确保有可见 loading/error/retry 状态。
- 收敛 HTML/PDF 回链行为，保证定位语义一致并提供 HTML 失败时降级路径。
- 修复 PDF 页码基准不一致导致的上下文定位错误风险。

**Non-Goals:**
- 不改动 LLM 卡片提取策略与提示词。
- 不重构整套导航结构（保持当前词表/复习页入口）。
- 不引入离线 HTML 解析与复杂预渲染缓存。

## Decisions

### 决策 1：词表采用“分组锚点 + 侧边索引栏”而非独立排序菜单
- 方案：按 term 首字符将词卡分组为 `A-Z` 与 `#`，列表使用可滚动锚点；右侧索引栏监听按压/滑动并触发滚动到分组首项。
- 原因：该交互符合 iOS 通讯录式心智模型，单手可达，定位速度优于频繁切换排序条件。
- 备选方案：顶部排序菜单（按字母/导入时间）被拒绝，因二次操作成本更高，无法满足“滑到 B 立刻定位”需求。

### 决策 2：原文阅读采用统一状态容器（loading / success / failure）
- 方案：在打开 HTML/PDF 时先进入统一加载页，成功后渲染阅读器，失败则展示错误文案与重试按钮。
- 原因：当前 Web/PDF 直接加载导致“空白等待”体验差；统一状态容器可同时覆盖词表与复习页入口。
- 备选方案：仅在按钮旁显示小菊花被拒绝，因无法覆盖全屏阅读器初始化阶段。

### 决策 3：HTML 不可用时可预测降级到 PDF
- 方案：当 HTML 导航失败（网络/404/失效）时，提示失败并提供“改用 PDF 打开”；若已知 PDF 可用可自动降级。
- 原因：arXiv HTML 可用性不稳定，必须有业务级兜底路径。
- 备选方案：失败后停留错误页被拒绝，恢复路径过长。

### 决策 4：统一 PDF 页码语义为阅读器可直接消费的 0-based
- 方案：在导入构建 anchor 时统一转换，保证 `PDFReaderView` 接收的 `targetPage` 与 `PDFDocument.page(at:)` 对齐。
- 原因：现有链路存在 1-based/0-based 混用风险，直接导致上下文定位错误。
- 备选方案：在阅读器层动态猜测页码被拒绝，隐式修正难以测试且易回归。

## Risks / Trade-offs

- [风险] 字母索引在小屏上触控区域过窄，误触概率上升。  
  → Mitigation：扩大手势命中区域，切字母时显示大号悬浮提示。
- [风险] 词条首字符归类规则（符号、重音字符）不一致导致定位预期差。  
  → Mitigation：定义稳定归类规则（A-Z，不匹配归 `#`）并写入 spec。
- [风险] HTML 自动降级 PDF 可能掩盖真实网络问题。  
  → Mitigation：在降级前显示可见原因，并保留重试 HTML 入口。
- [权衡] 增加状态管理复杂度（多入口共享）。  
  → Mitigation：抽取统一 Reader 状态模型，减少视图分支重复。

## Migration Plan

1. 先引入词表分组数据结构与索引 UI（不改变现有卡片数据模型）。
2. 引入统一阅读状态容器并接入 `SourceDetailView`，保证词表入口可用。
3. 复用同一容器到复习页“回到原文”入口。
4. 修正并验证 PDF 页码语义，添加回归用例覆盖首/中/末页。
5. 灰度验证：弱网、HTML 404、PDF 成功、PDF 无句子匹配等场景。

回滚策略：保留原阅读器组件，若新容器出现阻塞可切回“直接打开阅读器”路径。

## Open Questions

- HTML 失败后是否默认自动降级，还是总是先让用户点“改用 PDF”？
- 字母索引是否需要在词条较少时自动隐藏（例如 < 20 条）？
- `#` 分组是否需要单独展示标题，还是仅作为索引跳转目标？

# 项目命名与 Agent 开发提示词

## 推荐项目名

我建议主项目名使用 **PaperLoop**。[web:184][web:187][web:190]

这个名字适合你的原因是：
- **Paper** 直接对应论文 / PDF 阅读场景，定位清晰。[web:190]
- **Loop** 对应你的核心闭环：导入 paper → 自动抽词句 → 生成卡片 → 复习 → 回到原文。[web:108][web:114][web:124][web:131]
- 名字短、易记、适合 App Store 展示，也符合“简单、易拼写、暗示用途”的命名原则。[web:184][web:187][web:190]

### 备选名

- **PaperCue**：强调提示、提醒、学习线索，适合轻巧学习产品。[web:184][web:187]
- **SourceCard**：强调卡片与来源回链，但学术阅读感略弱。[web:124][web:190]
- **TermFlow**：强调术语与学习流，但不如 PaperLoop 直观对应 paper 场景。[web:184][web:187]
- **ReadBack**：强调回链原文，但更像阅读器功能名而不是完整产品名。[web:124][web:131]

### 推荐品牌表达

- 产品名：**PaperLoop**。[web:190]
- 副标题：**Turn papers into reviewable vocabulary and sentence cards.**[web:108][web:182]
- 中文名可用：**论文词句环** / **论文回学** / **PaperLoop 论文词句助手**。

## 给 Claude Code / Agent 的总提示词

下面是一份可以直接给 Claude Code、Cursor、Windsurf、Gemini CLI 等 Agent 使用的项目级提示词。结构重点参考了“先定义产品与范围，再明确交付物”的 Agent 工作流思路。[web:183][web:186][web:192]

### 通用提示词

你现在是资深产品设计师 + iOS 产品工程师 + AI MVP 架构师，请帮助我构建一个名为 **PaperLoop** 的 iPhone 优先学习应用 MVP。[web:183][web:189][web:190]

#### 项目背景

PaperLoop 是一个面向英文论文（paper / PDF）阅读场景的学习应用。用户输入 PDF 链接或上传本地 PDF 后，系统自动提取正文内容，识别值得学习的单词、术语短语和关键整句，并自动生成可复习的词卡和句卡。[web:108][web:110][web:182]

这个产品的核心不是“手动做 flashcards”，而是“把论文阅读过程自动沉淀为可复习资产”。系统需要支持单词发音、整句朗读、自动词表生成，以及从卡片回到 PDF 原文上下文的回链能力。[web:107][web:114][web:124][web:131][web:172]

#### MVP 目标

请只做 **iPhone MVP**，不要扩展成完整桌面端或复杂 iPad 工作台。iPhone 版只保留最关键的高频动作，避免功能过重。[web:141][web:144][web:148]

MVP 必须覆盖以下闭环：
1. 导入 PDF 链接或本地 PDF。[web:108][web:110]
2. 自动提取正文并自动生成候选词、术语、句卡。[web:107][web:114][web:168]
3. 生成自动词表，而不是要求用户手动逐个添加。[web:5][web:182]
4. 提供复习页，支持单词发音、整句朗读、显示答案和记忆反馈。[web:163][web:172][web:173]
5. 每张卡片必须保留来源信息，并支持“回到原文”。[web:124][web:126][web:131]
6. 提供词表页与基础设置页。[web:142][web:155]

#### 目标用户

- 需要频繁阅读英文 paper 的本科生、研究生、博士生、研究助理。
- 想从自己正在读的论文中学习术语、表达和句型，而不是背通用词表的用户。[web:182][web:185]

#### 核心价值主张

- 自动从 paper 里抽出“值得学”的内容，而不是让用户手工整理。[web:5][web:182]
- 词、术语、整句并重，而不是只做单词卡。[web:182][web:188]
- 每张卡都能回到原文页码、原句和上下文，建立阅读闭环。[web:124][web:126][web:131]
- iPhone 端强调高频复习、听读和轻量查看，而不是长时间复杂阅读。[web:142][web:148][web:172]

#### 信息架构

App 使用底部 Tab Bar，只有 4 个顶层入口：[web:142][web:149][web:151]
- 导入
- 复习
- 词表
- 我的

#### 5 个核心页面

1. **首页 / 导入页**
- 粘贴 PDF 链接
- 上传本地 PDF
- 显示预计生成的卡片数、术语数、关键句数
- 导入成功后进入解析状态，再跳转到复习页或待确认页

2. **今日复习页**
- 展示当前卡片
- 支持播放单词发音或整句音频
- 支持显示答案
- 支持四档反馈：不认识 / 模糊 / 认识 / 已掌握
- 支持从卡片跳去“回链详情”

3. **回链详情页**
- 显示来源 paper 名称、页码、段落位置
- 显示原句高亮
- 显示前后 1–2 句上下文
- 提供“回到原文”主按钮
- 保留“播放发音”“加入复习”“标记已掌握”等次级动作

4. **词表页**
- 自动生成的词表列表
- 支持按类别筛选：高频学术词、领域术语、句型、已掌握
- 每项显示出现次数、来源 paper 数、是否可播放音频
- 点击进入详情或回链详情

5. **我的页**
- 学习统计
- 导出（Anki / CSV）
- 设置：字体大小、朗读速度、深色模式、自动导词规则
- 账号与同步状态

#### 自动词表生成规则

请按以下原则设计自动抽取逻辑：
- 优先抽取高价值学术词和领域术语，而不是普通常见词。[web:182][web:185]
- 支持识别术语短语，不只识别单个单词。[web:182][web:188]
- 支持识别关键整句，重点是定义句、方法句、结果句、贡献句。[web:108][web:117]
- 自动去重归并相同词条、词族和重复术语。[web:114][web:182]
- 默认是“自动生成 + 用户轻编辑”，而不是纯手动添加。[web:5][web:182][web:189]

#### 回链数据结构要求

每张卡必须至少保留这些来源字段：
- paper_id
- paper_title
- source_page
- paragraph_index
- sentence_index
- selected_text
- source_sentence
- context_before
- context_after
- source_anchor_id 或 text_offset

当用户点击“回到原文”时，系统应根据这些字段打开对应 PDF，并定位到原句附近区域，同时高亮原句或选中文本。[web:124][web:131][web:174]

#### iOS 设计要求

请按 iPhone 原生内容型应用来设计，而不是网页缩放版。设计必须遵守以下原则：
- 使用底部 Tab Bar 作为顶层导航。[web:142][web:149][web:151]
- 主要交互目标尽量满足 44pt 左右触控尺寸。[web:88][web:89][web:160]
- 支持 Dynamic Type，避免长句在大字号下溢出或断裂。[web:87][web:89][web:157]
- 视觉风格克制、文本优先、中性色背景、低饱和主色点缀。[web:148][web:184]
- 音频按钮必须显性可见，因为发音和整句朗读是产品差异点。[web:172][web:173]

#### 视觉风格

请采用以下风格：
- 类似高质量内容型 iOS 应用
- 中性色背景（米白 / 灰米色）
- 深色正文文字
- 低饱和 teal 作为主强调色
- 大标题可略带编辑感，但整体不花哨
- 不要使用典型 AI 产品常见的紫色发光渐变或夸张插画

#### 功能非目标

请不要在 MVP 中加入以下内容：
- 社区功能
- 复杂笔记编辑器
- 多人协作
- 完整论文管理平台
- 完整 iPad 双栏版
- 浏览器插件

#### 技术实现建议

如果你要开始编码，请优先输出：
1. 产品信息架构。
2. 页面路由结构。
3. 数据模型定义。
4. 假数据 / mock 数据。
5. iPhone 高保真界面。
6. 可点击原型。
7. 再逐步接入 PDF 解析、TTS、自动抽词逻辑。[web:183][web:189][web:192]

#### 你输出时必须包含

- 清晰的页面列表
- 每个页面的主要组件
- 数据模型
- 关键用户流程
- 状态流转（导入中 / 解析完成 / 空词表 / 复习完成 / 回链失败）
- 一份可运行的前端 MVP

#### 交付标准

MVP 必须让用户完成以下真实链路：
“导入一篇 paper → 自动获得词表和句卡 → 在 iPhone 上复习一张卡 → 点击回到原文 → 查看上下文 → 返回继续复习”。[web:108][web:114][web:124][web:131]
